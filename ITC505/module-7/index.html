HTTP Status Codes

  • 205 Reset Content

    This status code indicates that the server has fulfilled the request and the client should reset the document view.

    Situation: A server responds with this code after a successful PUT request to inform the client that it should reset its view of the document.

  • 302 Found

    This status code indicates that the requested resource resides temporarily under a different URI. The client should follow the redirection provided in the Location header.

    Situation: A user is redirected to a different page or resource temporarily, such as during a login process or when a URL has been changed.

  • 405 Method Not Allowed

    This status code indicates that the request method is not allowed for the resource. The server should include an Allow header specifying the allowed methods.

    Situation: A client attempts to use a POST method on a resource that only supports GET requests, resulting in this error code.

  • 511 Network Authentication Required

    This status code indicates that the client needs to authenticate to gain network access. It is often used in captive portals.

    Situation: A user connects to a Wi-Fi network and is redirected to a login page where they need to authenticate before gaining access to the internet.

  • 411 Length Required

    This status code indicates that the server refuses to accept the request without a defined Content-Length header.

    Situation: A client sends a request with a method that requires a body, but the request lacks a Content-Length header, resulting in this error code.
